# Security Policy

OpenTopology is a specification and schema project. Security reports usually
concern ambiguity, misleading conformance behavior, validation bypasses, or
examples that encourage unsafe implementation.

## Report a Vulnerability

Please report security-sensitive issues privately to the project maintainers.

Current contact:

- Stuart Preston (stuart@pendrica.com)

If a public issue is already open, avoid posting exploit details until
maintainers have reviewed the report.

## Scope

In scope:

- schema behavior that allows invalid or misleading manifests
- conformance language that creates unsafe implementation ambiguity
- examples that teach insecure or deceptive usage
- reference tooling vulnerabilities, if tooling is added

Out of scope:

- vulnerabilities in third-party implementations
- product-specific deployment issues
- general support requests

## Handling

Maintainers will acknowledge reports, assess impact, and coordinate a fix or
public clarification when appropriate.
