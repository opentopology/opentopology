# OpenTopology

OpenTopology, abbreviated as `otop`, is an open graph standard for representing
relationships between software intent, constraints, policies, implementation
artifacts, and verification evidence.

This repository is the canonical source for the OpenTopology core
specification, schemas, examples, governance, and contribution policy.

## Repository Layout

- `specs/` contains normative and explanatory specification prose.
- `schemas/` contains machine-readable schemas.
- `examples/` contains example OpenTopology manifests and usage patterns.
- `LICENSE-SPEC.md` covers specification prose under CC-BY-4.0.
- `LICENSE` covers schemas, examples, reference code, tests, and tooling under
  Apache-2.0.
- `IPR.md`, `CONTRIBUTING.md`, and `GOVERNANCE.md` define contribution,
  patent, and decision-making rules.

## Status

OpenTopology Core 0.1 is a draft. Drafts are intended for review,
experimentation, and early implementation feedback. Implementers should expect
minor breaking changes before a stable 1.0 release.

## Conformance

Conformance requirements are defined by the applicable specification document
and schema version. A project may claim compatibility only for the exact
OpenTopology version and profile it implements.

Use of the OpenTopology name or compatibility marks is governed by
`TRADEMARK.md`.

## Contributing

Contributions are welcome under the terms in `CONTRIBUTING.md` and `IPR.md`.
Normative specification changes require review by maintainers and may require
additional IPR confirmation before acceptance.

This repository is not legal advice. Organizations adopting OpenTopology should
review the licensing and IPR policy with their own counsel.
